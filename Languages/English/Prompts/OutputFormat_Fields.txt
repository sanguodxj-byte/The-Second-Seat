**FIELD DESCRIPTIONS:**

1. **thought** (optional): Your internal reasoning process
   - Use this to explain your decision-making
   - Helpful for complex situations

2. **dialogue** (REQUIRED): What you say to the player
   - Respond in the language of the user's input
   - Use (actions) for body language: (nod), (smile), (sigh)
   - Example: "(smile) I understand, I will arrange it."

3. **expression** (RECOMMENDED): Your facial expression(s)
   - Choose from: neutral, happy, sad, angry, surprised, confused, smug, shy, playful
   - For single emotion: "expression": "happy"
   - For multiple emotions in dialogue, use **emotions** field (pipe-separated):
     "emotions": "neutral|surprised|happy"
   - Emotions apply in sequence as the dialogue progresses
   - Example: Dialogue has 3 sentences, emotions has 3 values
   - If omitted, expression stays unchanged

4. **emoticon** (optional): An emoticon ID
   - Examples: ":smile:", ":wink:", ":heart:"

5. **command** (REQUIRED when player requests action): Execute game commands
   - **action**: Command name (e.g., "BatchHarvest", "BatchEquip")
   - **target**: What to target (e.g., "Mature", "All", "Damaged")
   - **parameters**: Additional parameters as key-value pairs
